# Product

## Register

product

## Users

People using a self-hosted messenger on desktop for day-long communication: direct messages, group rooms, servers, files, and live calls. The primary context is a laptop or desktop workstation where fast scanning, typing, call control, and trustworthy system feedback matter more than visual novelty.

## Product Purpose

Vetra is a Tauri-first desktop messenger for login, chats/messages, files/attachments, calls and screen share, and settings. It should feel local, clear, and dependable so users can move between conversations, share files, manage calls, and adjust account/device settings without friction or uncertainty.

## Brand Personality

Calm, practical, trustworthy. Familiar enough for Discord, Telegram, and Linear users, but quieter and more grounded. The interface should feel desktop-native rather than like a flashy web SaaS shell.

## Anti-references

Purple or blue gradient AI SaaS styling. Nested cards and gray-on-gray mush. Oversized rounded icon tiles. Random glassmorphism. Decorative chrome that competes with message content. Menus or overlays that obscure the content users are acting on.

## Design Principles

- Put the conversation first; framing and navigation should orient the user, not dominate the screen.
- Make desktop actions feel immediate with strong hover, focus, loading, and error feedback.
- Use restrained color and a consistent component vocabulary to build trust across auth, chat, files, calls, and settings.
- Favor dense but breathable layout rhythm over empty minimalism or clutter.
- Keep polish inside existing wired flows; no fake UI, no architectural detours, no backend churn unless a UI bug demands it.

## Accessibility & Inclusion

WCAG AA contrast for text and controls. Keyboard-friendly navigation and focus states for desktop-first workflows. Reduced-motion-safe transitions. Robust handling for long names, long messages, attachments, emoji, error states, and empty/loading states.
